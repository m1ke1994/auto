# Vue Admin: Целевой Дизайн и Архитектура

## Статус
- Документ фиксирует целевое направление.
- Frontend (Vue/Nuxt), верстка, компоненты и интеграции на этом этапе не реализуются.

## Цель панели
Vue Admin — клиентская панель владельца сайта в SaaS-платформе.

Клиент видит только:
- свой сайт;
- разделы своего сайта;
- медиафайлы своего сайта;
- заявки своего сайта;
- настройки своего сайта;
- подписку своего аккаунта.

Клиент не видит:
- Django Admin;
- другие сайты;
- чужие данные;
- внутреннюю техническую структуру платформы.

## Визуальное направление
- Основная рабочая зона: светлая.
- Sidebar: темный.
- Accent: фиолетово-синий.
- Карточки: мягкие тени, rounded corners.
- Типографика: чистая и современная SaaS.
- Общий стиль: premium, аккуратный dashboard.
- Обязательная адаптивность для мобильных устройств.

## Архитектурный принцип данных
Vue Admin строится только на Django API. Хардкод структуры разделов запрещен.

Источник разделов:
- `GET /api/client/sections/`

Источник формы редактирования секции:
- `GET /api/client/sections/<slug>/form/`

Принцип form engine:
- `schema` описывает поля.
- `content` хранит значения.
- UI строится динамически: `schema -> форма`, `content -> значения`.

## Поддерживаемые типы полей (для динамических форм)
- `text`
- `textarea`
- `image`
- `video`
- `number`
- `boolean`
- `select`
- `repeater`

## Ограничения безопасности в клиентской панели
- Клиентский контур работает только с данными текущего пользователя.
- Доступ к чужим сайтам/секциям/медиа запрещен.
- Изменение системных полей секций запрещено, редактируется только допустимый контент согласно backend-правилам.
