# MVP: QA и Чек-лист Пользователя

## 1. Подготовка окружения

- [ ] В `.env` установлено: `ENABLE_BILLING=false`
- [ ] Применены миграции:
  - `python manage.py migrate`
- [ ] Создан тестовый владелец и связь с сайтом:
  - `python manage.py seed_amedia_owner`
- [ ] Запущены backend и Vue Admin:
  - Backend: `python manage.py runserver`
  - Vue Admin: `cd vue-admin && npm run dev`

Тестовые данные:
- Email: значение `AMEDIA_OWNER_EMAIL`
- Пароль: значение `AMEDIA_OWNER_PASSWORD`
- Сайт: `A Meditation / Амедиа`

## 2. QA Чек-лист (ручная проверка в браузере)

### 2.1 Авторизация и доступ к сайту

- [ ] Открыть Vue Admin: `http://localhost:5173/login`
- [ ] Войти с учётными данными из `AMEDIA_OWNER_EMAIL` и `AMEDIA_OWNER_PASSWORD`
- [ ] Пользователь видит только 1 сайт: `A Meditation / Амедиа`
- [ ] Ошибка `Client dashboard access is available only for active client users.` не появляется

### 2.2 Подписка и доступы MVP

- [ ] Перейти в `Mini CRM` (пункт меню `Mini CRM`)
- [ ] Не отображается блокирующее предупреждение по подписке
- [ ] Проверить разделы:
  - [ ] Обзор
  - [ ] Заявки
  - [ ] SEO-аудит
  - [ ] Отчеты
  - [ ] Настройки
  - [ ] Интеграция

### 2.3 SEO-аудит

- [ ] Открыть `Mini CRM -> SEO-аудит`
- [ ] Нажать `Запустить аудит`
- [ ] Аудит создается без ошибки подписки
- [ ] Открываются карточка аудита, история и AI-рекомендации
- [ ] Экспорт PDF из аудита выполняется успешно

### 2.4 Заявки

- [ ] Создать тестовую заявку через публичную форму/endpoint
- [ ] В `Mini CRM -> Заявки` новая заявка видна
- [ ] В Django Admin `/admin/leads/lead/` заявка также видна

### 2.5 Отчеты и PDF

- [ ] Открыть `Mini CRM -> Отчеты`
- [ ] Включить/выключить ежедневный PDF
- [ ] Сформировать отчет вручную (если кнопка доступна)
- [ ] PDF открывается, русский текст отображается без проблем кодировки

### 2.6 Telegram

- [ ] Открыть `Mini CRM -> Настройки`
- [ ] Поля Telegram доступны (статус, connect URL, флаг отправки)
- [ ] Переключатель `send_to_telegram` сохраняется
- [ ] Открыть `Mini CRM -> Интеграция` и проверить блок интеграции

## 3. Чек-лист Пользователя (как пользоваться приложением)

### 3.1 Первый вход

- [ ] Откройте `http://localhost:5173/login`
- [ ] Введите email и пароль
- [ ] После входа выберите свой сайт (или откроется автоматически, если он один)

### 3.2 Разделы админки сайта

- [ ] `Разделы`:
  - Редактируйте контент блоков сайта
  - Сохраняйте изменения и проверяйте на фронтенде
- [ ] `Аналитика`:
  - Смотрите визиты, уникальных пользователей, конверсию
- [ ] `Заявки`:
  - Просматривайте входящие лиды и меняйте статусы

### 3.3 Mini CRM

- [ ] `Обзор`:
  - Ключевые KPI по сайту
- [ ] `Заявки`:
  - Работа с лидами в mini-интерфейсе
- [ ] `SEO-аудит`:
  - Запуск аудита домена
  - Просмотр проблем, истории и рекомендаций
  - Экспорт PDF отчета
- [ ] `Отчеты`:
  - Управление ежедневным PDF
- [ ] `Настройки`:
  - Telegram-уведомления
  - Клиентские настройки
- [ ] `Интеграция`:
  - API key, tracker script и примеры подключения

### 3.4 Ежедневный рабочий сценарий

- [ ] Проверить новые заявки
- [ ] Обновить статусы лидов
- [ ] Проверить обзор/аналитику
- [ ] При необходимости запустить SEO-аудит
- [ ] Сформировать или отправить PDF отчет

## 4. Проверка production-поведения подписок

Перед включением платных ограничений:
- [ ] Установить `ENABLE_BILLING=true`
- [ ] Проверить, что при неактивной подписке закрываются только платные функции
- [ ] Проверить, что при активной подписке доступ восстанавливается
